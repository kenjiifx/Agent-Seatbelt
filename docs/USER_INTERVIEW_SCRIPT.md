# User Interview Script

## Audience

- Developers using Cursor / Claude Code / Codex / Gemini CLI
- Security engineers at startups
- DevOps/platform engineers
- Open-source maintainers

## Questions

1. Which coding agents do you use today?
2. Do those agents run terminal commands in your workflow?
3. Have agents ever touched secrets or sensitive files?
4. Have agents broken anything in your repo/system?
5. Which command classes should always be blocked?
6. Which command classes should require approval?
7. Would you use a local CLI runtime guardrail?
8. Would your team pay for team policy/audit sync?
9. What would stop you from adopting this next week?
10. What integration matters most (IDE, CLI, CI, MCP)?

## Interview outcomes to record

- Risk incidents mentioned
- Required policy controls
- Must-have integrations
- Pricing sensitivity (if discussed)
- Pilot interest (yes/no)
