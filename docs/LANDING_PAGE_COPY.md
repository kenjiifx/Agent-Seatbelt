# Landing Page Copy

## Hero

Runtime security for AI coding agents.

## Subhero

AgentSeatbelt intercepts risky terminal actions before execution, enforces local policy, blocks secret reads, requests approval for dangerous commands, and writes tamper-evident audit receipts.

## Section: The risk moved from suggestions to execution

AI coding agents increasingly execute commands, mutate repos, and trigger deploy paths. Runtime controls are now required.

## Section: How AgentSeatbelt works

Classify risk -> apply policy -> require approval or block -> write receipt -> verify chain.

## Section: What it blocks

- Secret reads (`.env`, keys, tokens)
- Destructive file operations
- Dangerous deploy/cloud mutations
- High-risk git operations

## Section: Local-first by default

- No telemetry
- No cloud upload by default
- Deterministic policy engine

## Section: Built for developers, useful for security teams

Developer-first CLI wedge with a path toward team/runtime governance.

## Section: Demo

Run the 90-second demo script and review receipts + verification output.

## Section: Roadmap

CLI wedge -> agent integrations -> CI/MCP enforcement -> team policy sync -> org governance.
